    <p><span>Для описания правил синтаксиса языка программирования применяются формализованные системы обозначений – метаязыки. Существует два основных метаязыка: </span></p>
    <ul>

        <li><span>Синтаксические диаграммы.</span></li>
    </ul>
    <p><span>Язык РБНФ более строг и точен, более удобен для представления синтаксиса в памяти машины, более компактен. Синтаксические диаграммы же более громоздки, однако намного нагляднее и проще для понимания.</span></p>
    <p><span>        Данная курсовая работа посвящена разработке программного средства для построения синтаксических диаграмм с использованием векторной графики. </span></p>

            <h2 id="h.3znysh7" style="display:inline"><span>Описание синтаксиса языка с помощью синтаксических диаграмм</span></h2>
        </li>
    </ol>
    <p><span></span></p>
    <p><span>Синтаксическая диаграмма позволяет графически изобразить структуру синтаксической единицы.</span></p>
    <p><span>        В метаязыках, описывающих синтаксис языка программирования, используются следующие основные понятия:</span><span>[1]</span></p>
    <ul>
        <li><span>Метапеременная – </span><span>обозначает определенную синтаксисом конструкцию языка. Для записи метапеременных в основном используются последовательности слов на естественном языке (русский, английский или др.) и служебных слов. Для разделения слов используется символ нижнего подчеркивания (_). В синтаксических диаграммах метапеременные заключаются в угловые скобки (<>). Метапеременная на размеченном ребре графа означает, что этот фрагмент диаграммы должен быть детализирован подстановкой синтаксической диаграммы с именем, соответствующим данной метапеременной. </span></li>
    </ul>
    <p><span>Примеры записи метапеременных:</span></p>
    <p><span><Оператор_For></span></p>
    <p><span><Тип_Set></span></p>
    <p><span><Базовый_скалярный_тип></span></p>
    <ul>
        <li><span>Метаконстанты – </span><span>обозначает лексему языка программирования. В программе метаконстанте соответствует она сама. В синтаксических диаграммах метаконстанты записываются «как есть».</span></li>
    </ul>
    <p><span>Примеры метаконстант:</span></p>
    <p><span>For</span></p>
    <p><span>Begin</span></p>
    <p><span>Set</span></p>
    <ul>
        <li><span>Метасимвол</span><span> – специальный символ, используемый для описания синтаксиса языка. В синтаксических диаграммах присутствует два единственных метасимвола:</span></li>
    </ul>
    <ul>
        <li><span>Метасимвол “::=” - используется для отделения имени синтаксической диаграммы. </span></li>
        <li><span>Метасимвол “<>” – используется для обозначения метапеременных</span></li>
    </ul>
    <p><span>Синтаксическая диаграмма представляет собой ориентированный граф с размеченными ребрами. Для разметки ребер используются метаконстанты и метапеременные. </span></p>

        <li>
            <h2 id="h.1t3h5sf" style="display:inline"><span>Представление векторного изображения в формате SVG</span></h2>
        </li>
    </ol>
    <p><span></span></p>
    <p><span>Векторная графика – это способ представления изображений и объектов, основанный на математическом описании элементарный географических объектов. </span></p>
    <p><span>Одним из самых распространенных форматов файлов векторной графики является формат SVG. Формат SVG предназначен для описание двумерной векторной и смешанной графики в текстовом формате XML.</span></p>
    <p><span>Структура документа</span><span>[2][3]</span><span>:</span></p>
    <ul>
        <li><span>Первая строка – стандартный XML заголовок с указанием версии, кодировки. </span></li>
    </ul>
    <p><span>Пример:</span></p>
    <p><span><?xml version="1.0" encoding="UTF-8" standalone="no"?></span></p>
    <ul>
        <li><span>Вторая и третья строка – заголовок DOCTYPE, определяющий тип документа.</span></li>
    </ul>
    <p><span>Пример:</span></p>
 <div class="highlight highlight-source-xml">   
   <pre> 
    &lt;!DOCTYPE svg PUBLIC &quot;-//W3C//DTD SVG 1.1//EN&quot;
   &quot;http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd&quot;&gt;
   </pre>
   </div>
    <ul>
        <li><span>Четвертая строка – корневой элемент документ с указанием пространства имен SVG</span></li>
    </ul>
    <p><span>Пример:</span></p>
<pre>&lt;svg version=&quot;1.1&quot;
     baseProfile=&quot;full&quot;
     xmlns=&quot;http://www.w3.org/2000/svg&quot;
     xmlns:xlink=&quot;http://www.w3.org/1999/xlink&quot;
     xmlns:ev=&quot;http://www.w3.org/2001/xml-events&quot;
     width=&quot;100%&quot; height=&quot;100%&quot;&gt;
     </pre>
    <ul>
        <li><span>Далее идет остальной текст документа, завершающийся закрытием тега </span><span><pre>&lt/svg></pre></span></li>
    </ul>
    <p><span><br><br></span></p>
    <p><span>Отрисовка основных фигур:</span></p>
    <ul>
        <li><span>Описание путей</span></li>
    </ul>
    <p><span>Позволяет задать любую фигуру, описывая путь от начальной точки до конечной через промежуточные координаты. Строка с данными задается атрибутом d тега </span><span>path</span><span> и содержит команды, закодированные набором букв и чисел. Буквы – обозначают тип команды. Наиболее простые – M </span><span>(англ. moveto – переместить)</span><span>, L </span><span>(англ. lineto – нарисовать линию). </span><span>Цифры, чаще всего, содержат координаты точек по осям X и Y.</span></p>
    <p><span>Пример: линия из точки (100,100) в точку (100,200)</span></p>
    <p><span><pre>&lt;path fill=&quot;none&quot; stroke=&quot;black&quot; d=&quot;M 100 100 L 100 200&quot; /&gt;
    </pre></span></p>
    <ul>
        <li><span>Прямоугольник</span></li>
    </ul>
    <p><span>Строка задается 4-мя основными атрибутами тега </span><span>rect</span><span>: координаты X,Y левой верхней точки (Атрибуты x, y), высота и ширина (Атрибуты height и width соответственно). </span></p>
    <p><span>Пример: </span></p>
    <p><span><pre>&lt;rect fill=&quot;white&quot; x=&quot;400&quot; y=&quot;600&quot; width=&quot;300&quot; height=&quot;200&quot; /&gt;
    </pre></span></p>
    <ul>
        <li><span>Окружность</span></li>
    </ul>
    <p><span>Строка задается 3-мя основными атрибутами тега </span><span>cricle</span><span>: координаты центра (Атрибуты cx, cy), радиус (Атрибут r).</span></p>
    <p><span>Пример:</span></p>
    <p><span><pre>&lt;circle cx=&quot;200px&quot; cy=&quot;200px&quot; r=&quot;104px&quot; fill=&quot;red”/&gt;
    </pre> </span></p>
    <ul>
        <li><span>Вывод текст</span></li>
    </ul>
    <p><span>Выводимый текст заключается в тег </span><span>text</span><span>,</span><span> </span><span>в котором в качестве атрибутов задаются свойства. Элементарные свойства для вывода текста – координаты левой верхней точки текста (атрибуты x, y).</span></p>
    <p><span>Пример: </span></p>
    <p><span><pre>&lt;text x=&quot;30&quot; y=&quot;12&quot; &gt;Syntax Diagrams Editor&lt;/text&gt;
    </pre></span></p>
    <p><span></span></p>
    <p><span>Каждому тегу можно задать дополнительные свойства, описания и примеры которых находятся в документации по формату SVG.</span></p>

    <p><span>         </span></p>

    <p><span>В зависимости от выбранного режима пользователю должны представляться следующие возможности взаимодействия:</span></p>
    <ul>
        <li><span>Режим рисования прямоугольных фигур:</span></li>
    </ul>
    <ul>
        <li><span>При клике по полотну программное средство должно отобразить введенный пользователем текст на канвасе по координатам клика. </span></li>
    </ul>
    <ul>
        <li><span>Режим рисования линий:</span></li>
    </ul>
    <ul>
        <li><span>Первый клик левой кнопкой мыши по полотну должен активировать режим рисования линий. Каждое следующее нажатие левой кнопки мыши должно добавлять новую точку и соединить ее с предыдущей точкой данной линии. Нажатие правой кнопки мыши должно прекратить рисование линии. </span></li>
    </ul>
    <ul>
        <li><span>Режим редактирования:</span></li>
    </ul>
    <ul>
        <li><span>При перемещении мыши курсор меняется в зависимости от того, на какую область фигуры он наведен:</span></li>
    </ul>
    <ul>
        <li><span>Вершина фигуры</span></li>
        <li><span>Сторона фигуры</span></li>
        <li><span>Центр фигуры</span></li>
    </ul>
    <ul>
        <li><span>При зажатии мыши в этом режиме, фигура должна редактироваться по следующему принципу</span></li>
    </ul>
    <ul>
        <li><span>Зажата в центре – при перемещении курсора перемещается вся фигура. </span></li>
        <li><span>Зажата на вершине – при перемещении курсора перемещается вершина и стороны, которым принадлежит вершина. </span></li>
        <li><span>Зажата сторона фигуры – при перемещении курсора перемещается сторона. </span></li>
    </ul>
    <ul>
        <li><span>Клик по фигуре должен выделять фигуру, по которой был произведен клик. При нажатии клавиши «delete» должна удаляться выделенная фигура. Сочетания «Ctrl + C» должно помещать в специальный «программный буфер обмена» выделенную фигуру. Сочетания «Ctrl + V» - извлекать из буфера фигуру и отображать на полотне. Сочетание «Ctrl + Z» должно «откатывать» на предыдущее действие, то есть отменять последнее изменение. Если фигура (не линия) выделена, то если изменить текст в отведенном текстовом поле и нажать «enter», текст внутри фигуры должен измениться на введенный. </span></li>
    </ul>
    <p><span>        Также программа должна предоставлять взаимодействие с меню. В зависимости от того, по какому элементу меню был произведен клик, программа должна уметь:</span></p>

        <li><span>Открывать исходный файл.</span></li>

    <p><span></span></p>
    <p><span>Модуль Model.Lines является дополнительным юнитом модели. В модуле присутствуют подпрограммы взаимодействия с линиями. </span></p>
    <p><span>Основная идея взаимодействия со стеком – каждое изменение пользователя добавляет запись в стек. Если пользователь хочешь отменить изменение – извлекается запись из вершины стека и обрабатывается в модели. </span></p>
    <p><span>Обработке поддаются следующие изменения:</span></p>
    <ul>
        <li><span>Удаление фигуры (В стеке содержится ссылка удаленной страницы и ссылка на фигуру, расположенную перед удаленной)</span></li>
        <li><span>Добавление точки линии (В стеке содержится ссылка на линию и ссылка на добавленную точку)</span></li>
        <li><span>Добавление фигуры (В стеке содержится ссылка на добавленную фигуру)</span></li>
        <li><span>Перемещение фигуры/изменение размеров фигуры (кроме линий) (В стеке содержится ссылка на фигуру, которую переместили и предыдущее значение записи информативной части фигуры (предыдущие координаты))</span></li>
        <li><span>Перемещение линии / перемещение точки линии (В стеке содержится ссылка на фигуру линии, которую переместили и копию старых координат всех точек, сохраненных в текстовом формате)</span></li>
        <li><span>Изменение текста фигуры (Кроме линий) (В стеке содержится ссылка на фигуру и предыдущий текст)</span></li>
        <li><span>Изменение размеров полотна (В стеке содержатся предыдущие значения ширины и высоты полотна)</span></li>
    </ul>
